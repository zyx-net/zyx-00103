# 稿件更正与发布审批工作台 - 产品需求文档

## 1. 产品概述

一个本地可启动的稿件更正与发布审批 Web 工作台，用于管理新闻稿件的更正流程。采用「记者→编辑→法务→管理员」的审批链路，支持稿件更正单的创建、提交、复核、确认和发布全流程。

- 解决新闻机构对已发布稿件的更正管理混乱问题
- 确保稿件更正经过规范的审批流程
- 提供完整的操作历史追溯
- 支持数据导出和筛选配置

**目标用户：** 记者、值班编辑、法务人员、管理员

---

## 2. 核心功能

### 2.1 用户角色

| 角色 | 用户名 | 密码 | 核心权限 |
|------|--------|------|----------|
| 记者 | journalist | journalist123 | 提交更正单、查看自己稿件、查看历史 |
| 值班编辑 | editor | editor123 | 复核更正单、退回修改、查看所有稿件 |
| 法务 | legal | legal123 | 确认来源争议、审核合规性、查看所有稿件 |
| 管理员 | admin | admin123 | 发布更正、撤销发布、管理配置、导出数据 |

### 2.2 功能模块

1. **登录页面** - 角色选择登录
2. **工作台首页** - 稿件列表、更正单状态筛选、快捷操作入口
3. **稿件管理** - 稿件列表、创建稿件、查看详情
4. **更正单管理** - 创建更正单、更正单详情、审批操作
5. **历史记录** - 所有操作的历史追溯
6. **配置管理** - 筛选配置、数据导出

## 3. 审批工作流

### 3.1 标准流程

```
记者提交 → 编辑复核 → 法务确认 → 管理员发布
     ↓          ↓          ↓          ↓
  [可退回]   [可退回]   [可退回]   [发布成功]
```

### 3.2 流程状态

| 状态 | 描述 | 可执行操作 |
|------|------|-----------|
| 待编辑复核 | 记者已提交，等待编辑复核 | 复核通过/退回 |
| 待法务确认 | 编辑已复核，等待法务确认 | 确认通过/退回 |
| 待发布 | 法务已确认，等待管理员发布 | 发布/退回 |
| 已发布 | 更正已发布上线 | 撤销 |
| 已退回 | 被退回修改 | 修改后重新提交 |

### 3.3 更正单信息

| 字段 | 说明 | 必填 |
|------|------|------|
| 稿件ID | 关联的原始稿件 | 是 |
| 更正类型 | 事实错误/标题错误/来源更正/内容补充/其他 | 是 |
| 证据说明 | 更正的依据和说明 | 是 |
| 截止时间 | 更正完成的截止日期 | 是 |
| 影响范围 | 受影响读者数量/范围描述 | 是 |
| 来源争议 | 是否涉及来源争议 | 是 |

## 4. 验证规则与错误处理

### 4.1 必填验证
- 缺少证据说明 → 提示"请填写证据说明"，不改状态
- 缺少截止时间 → 提示"请填写截止时间"，不改状态
- 缺少影响范围 → 提示"请填写影响范围"，不改状态

### 4.2 权限验证
- 记者越权审批 → 提示"记者无权执行此操作"，不改状态
- 编辑越权发布 → 提示"编辑无权发布更正"，不改状态
- 法务越权操作 → 提示"法务无权执行此操作"，不改状态

### 4.3 流程验证
- 来源争议未经过法务确认 → 提示"来源争议必须经过法务确认"，不改状态
- 重复发布 → 提示"该稿件已发布更正，不可重复发布"，不改状态
- 状态不符 → 提示"当前状态不允许此操作"，不改状态

### 4.4 数据约束
- 已发布更正只能撤销，不能删除
- 所有操作必须写入历史记录
- 重启后所有数据必须持久化保存

## 5. 用户界面设计

### 5.1 设计风格
- **主题：** 深色专业新闻风格，强调信息层次
- **主色调：** 深蓝 (#1a1a2e) 配橙色强调 (#f39c12)
- **字体：** 思源黑体 (Noto Sans SC) + JetBrains Mono
- **布局：** 侧边导航 + 主内容区，卡片式信息展示

### 5.2 页面设计

| 页面 | 模块 | UI 元素 |
|------|------|---------|
| 登录页 | 角色选择器 | 大图标卡片，悬停高亮，4个角色 |
| 工作台 | 状态统计卡片 | 数字统计，环形进度图 |
| 工作台 | 稿件列表 | 表格+卡片切换，状态标签筛选 |
| 工作台 | 待办事项 | 时间线样式，紧急程度颜色 |
| 更正单 | 表单 | 分步骤表单，实时验证 |
| 更正单 | 审批面板 | 操作按钮，历史时间线 |
| 历史 | 操作记录 | 时间线卡片，操作人头像 |

### 5.3 响应式设计
- 桌面端优先 (≥1024px)
- 平板适配 (768px-1024px)
- 移动端基础支持 (<768px)
